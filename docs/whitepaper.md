# Syndaxia Protocol — Whitepaper

**Version:** 1.1 (March 2026)  
**Entities:** Satflows SAS (Publisher) / Syndaxia Association (Governance)  
**Concept:** Decentralized trust orchestrator for P2P and B2B commerce.

---

## 1. Vision and Objectives

**Syndaxia** aims to build a secure, open payment infrastructure connecting buyers and sellers without any central financial intermediary.

- **Problem:** Lack of trust in peer-to-peer (P2P) transactions, and payment delays in B2B commerce.
- **Solution:** An automated escrow powered by smart contracts, where funds are locked until a delivery condition is met (physical delivery, QR code scan, time-based release).
- **Differentiator:** A fully non-custodial model — funds never pass through company accounts — keeping the protocol outside the scope of MiCA and PSD2 regulations.

---

## 2. Entity Structure

The project rests on three pillars designed for legal and technical resilience:

1. **Syndaxia Association (Non-profit, French Law 1901)**
   - Holds intellectual property over the protocol code.
   - Issues the governance token $SDX.
   - Ensures network neutrality.

2. **Satflows SAS (Commercial entity)**
   - Publisher of the Syndaxia Pay application.
   - Develops the user interface, manages banking partnerships (Monerium, Bridge, Wero), and monetizes the service.
   - Compensated through software service fees.

3. **The DAO (Community)**
   - $SDX token holders who vote on protocol parameters (fee rates, oracle selection).

---

## 3. Technical Architecture

| Component | Technology |
|---|---|
| Blockchain | Solana (speed, minimal fees) |
| Smart Contracts | Anchor 0.32.0 — singleton escrow program |
| Interface | Account abstraction (smart wallets) — email/Google login |
| On-ramp | Monerium, Bridge, Wero — fiat → EURC/USDC conversion |

Funds flow directly from the buyer's wallet to a **Program Derived Address (PDA)** controlled solely by the protocol code. Satflows holds no private key for these accounts.

---

## 4. Use Cases

### Target Audiences

- **C2C (Consumers):** Secure second-hand sales (marketplace-style, without a proprietary platform).
- **B2B (Professionals):** Invoice payments at 30/60 days with funds locked on-chain as guarantee.
- **Services:** Freelance engagements with funds released upon delivery of a milestone.
- **Agentic Commerce:** AI agents acting as autonomous economic entities — renting compute, purchasing data streams, or paying verification agents — where each micro-task requires trustless, instant settlement. Traditional banking (T+2) and custodial platforms are incompatible with machine-speed execution; Syndaxia's sub-second finality and immutable logic make it the natural settlement layer for agent-to-agent transactions.

### Release Conditions

| Mode | Trigger |
|---|---|
| Standard | Manual buyer approval after receipt |
| Physical | QR code scan generated by the seller |
| Logistics | API notification from a carrier (Colissimo, Mondial Relay) |
| Time-based | Automatic release after X days without dispute |

## 6. Compliance & Safety Guarantees (2026)

- **Zero custody:** Satflows SAS never holds user private keys or funds.
- **Technical independence:** The smart contract is either immutable or under DAO control (via the Association), not Satflows alone.
- **Dispute resolution:** On-chain freeze mechanism; a designated validator resolves disputes. No internal arbitration complexity in Phase 1.

---

## 7. Tokenomics ($SDX)

- **Utility:** Fee discounts for buyers, governance votes (oracle selection, fee rates), retroactive airdrop for early adopters.
- **Launch:** Issued by the Syndaxia Association after a closed beta validating real usage.
